# 🚨 紧急修复指南 - 数据库配置未生效

## 📊 问题诊断

根据浏览器开发者工具的错误信息：
- **错误代码：** 503 Service Unavailable
- **错误原因：** 后端服务运行中，但数据库配置未生效
- **根本原因：** 后端服务未重启，仍在使用旧配置

## ✅ 解决方案

### 方法 1: 使用自动重启脚本（推荐）

**步骤 1: 双击运行重启脚本**

找到文件：`G:\demo_1\demo\server\restart.bat`

双击运行该文件，它会自动：
1. 停止旧的后端服务
2. 启动新的后端服务
3. 加载最新的配置

**步骤 2: 等待启动完成**

看到以下信息表示启动成功：
```
服务器运行在端口 3000
```

**不应该看到：**
```
❌ Supabase 配置错误！
```

### 方法 2: 手动重启（如果脚本失败）

**步骤 1: 停止所有后端服务**

打开 PowerShell 或 CMD，运行：
```bash
# 查找占用 3000 端口的进程
netstat -ano | findstr :3000

# 记下最后一列的 PID（进程ID），然后结束进程
taskkill /PID <PID> /F

# 例如：taskkill /PID 12345 /F
```

**步骤 2: 重新启动后端服务**

```bash
cd G:\demo_1\demo\server
npm start
```

**步骤 3: 验证启动日志**

应该看到：
```
服务器运行在端口 3000
```

### 方法 3: 使用任务管理器（最简单）

**步骤 1: 打开任务管理器**
- 按 `Ctrl + Shift + Esc`

**步骤 2: 找到并结束 Node.js 进程**
- 在"进程"标签页中找到 "Node.js JavaScript Runtime"
- 右键点击 → "结束任务"

**步骤 3: 重新启动后端服务**
```bash
cd G:\demo_1\demo\server
npm start
```

## 🔍 故障排查

### 问题 1: 端口被占用

**错误信息：**
```
Error: listen EADDRINUSE: address already in use :::3000
```

**解决方案：**
```bash
# 查找并结束占用 3000 端口的进程
netstat -ano | findstr :3000
taskkill /PID <PID> /F

# 或者修改端口
# 编辑 server/.env 文件
PORT=3001
```

### 问题 2: 配置文件未找到

**错误信息：**
```
❌ Supabase 配置错误！
```

**解决方案：**
1. 确认 `.env` 文件位置：`G:\demo_1\demo\server\.env`
2. 确认文件内容：
   ```env
   SUPABASE_URL=https://uxanovqmqikbkuonttjo.supabase.co
   SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
   ```
3. 确认没有多余的空格或引号

### 问题 3: 数据库连接失败

**错误信息：**
```
Error: Invalid supabaseUrl
```

**解决方案：**
1. 访问 https://supabase.com
2. 确认项目存在
3. 确认 URL 和 Key 正确
4. 确认数据表已创建
