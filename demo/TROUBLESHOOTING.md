## 🎯 问题原因

**后端服务在您修改 `.env` 文件之前就已经启动了**，所以它还在使用启动时的环境变量（空值）。

Node.js 进程只在启动时读取一次环境变量，修改 `.env` 文件后必须重启服务才能生效。

## ✅ 解决方案

### 方法 1: 手动重启（推荐）

**步骤 1: 找到运行后端的终端窗口**

您应该有一个终端窗口显示：
```
服务器运行在端口 3000
```

**步骤 2: 停止后端服务**

在该终端窗口中按 `Ctrl + C`

**步骤 3: 重新启动后端服务**

```bash
cd server
npm start
```

**步骤 4: 验证配置**

启动后应该看到：
```
服务器运行在端口 3000
```

**不应该看到：**
```
❌ Supabase 配置错误！
```

### 方法 2: 使用新终端窗口

**步骤 1: 打开新的终端窗口**

**步骤 2: 启动后端服务**

```bash
cd G:\demo_1\demo\server
npm start
```

**步骤 3: 关闭旧的终端窗口**

（如果旧的后端服务还在运行，会提示端口被占用）

### 方法 3: 强制结束进程（如果上述方法无效）

**Windows 系统：**

```bash
# 查找占用 3000 端口的进程
netstat -ano | findstr :3000

# 结束进程（替换 PID 为实际的进程ID）
taskkill /PID <进程ID> /F

# 重新启动
cd server
npm start
```

## 📝 常见问题

### Q: 重启后还是显示"数据库未配置"

**A:** 检查以下几点：

1. **确认 `.env` 文件位置**
   - 应该在 `G:\demo_1\demo\server\.env`
   - 不是在 `G:\demo_1\demo\.env`

2. **确认 `.env` 文件内容**
   ```env
   SUPABASE_URL=https://uxanovqmqikbkuonttjo.supabase.co
   SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
   ```

3. **确认没有多余的空格或引号**
   - ❌ 错误: `SUPABASE_URL="https://..."`
   - ✅ 正确: `SUPABASE_URL=https://...`

### Q: 端口被占用

**A:** 修改端口号

编辑 `server/.env`:
```env
PORT=3001
```

然后重启服务，访问 `http://localhost:3001`

### Q: 前端无法连接后端

**A:** 更新前端配置

编辑 `client/.env`:
```env
VITE_API_BASE_URL=http://localhost:3000/api
```

（如果后端端口改了，这里也要改）
